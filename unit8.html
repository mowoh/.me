Unit 8: Refactoring and Code Smells

The module provided an Introduction to Code Smells:

Objectives

  • Understand and explain what code smells are and how to detect them in a codebase.
  • Apply refactoring techniques to improve code maintainability and readability.
  • Refactor an existing codebase to eliminate code smells, using object-oriented principles.
  • Recognise the importance of refactoring in maintaining high-quality, sustainable software systems.

    • Task

    A monolithic code containing vulnerablities like use of magic numbers, no input validation, does not follow OOP, mixes responsiblity 
    
# Non-OOP version

employees = []

def add_employee(name, hourly_rate, hours_worked):
    employee = {
        "name": name,
        "hourly_rate": hourly_rate,
        "hours_worked": hours_worked
    }
    employees.append(employee)

def calculate_pay(employee):
	if employee["hours_worked"] < 40:
		return None
    return employee["hourly_rate"] * employee["hours_worked"]

def print_payslip(employee):
    pay = calculate_pay(employee)
    print(f"Employee: {employee['name']}")
    print(f"Hours Worked: {employee['hours_worked']}")
    print(f"Hourly Rate: {employee['hourly_rate']}")
    print(f"Gross Pay: {pay}")

add_employee("Alice", 25, 40)
add_employee("Bob", 30, 35)

for employee in employees:
    print_payslip(employee)

    Solution Steps

  • 1. Converted procedural code into object-oriented design
    The original version used:
    • a global employees list
    • dictionary-based employee records
    • standalone functions such as add_employee(), calculate_pay(), and print_payslip()
    This was refactored into classes:
    • Employee
    • PayrollSystem
    This grouped related data and behavior together.
    2. Replaced dictionaries with class instances
    Instead of storing employee data in dictionaries, each employee became an Employee object with:
    • name
    • hourly_rate
    • hours_worked
    This improved encapsulation and made the code easier to understand.
    3. Moved behavior into methods
    The original standalone functions were moved into class methods:
    • calculate_pay() became Employee.calculate_pay()
    • print_payslip() became Employee.print_payslip()
    • employee collection management moved into PayrollSystem.add_employee()
    This made responsibilities clearer.
    4. Introduced overtime logic
    The OOP version was extended to support overtime pay calculation:
    • standard hours were treated differently from overtime hours
    • overtime pay used a multiplier
    This added more realistic payroll behavior.
    5. Introduced magic numbers for demonstration
    In the intermediate OOP version, hard-coded values were added:
    • 40 for standard working hours
    • 1.5 for overtime multiplier
    These were included intentionally to demonstrate the problem of magic numbers.
    6. Refactored magic numbers into named constants
    The hard-coded values were then replaced with named class constants:
    • STANDARD_HOURS
    • OVERTIME_MULTIPLIER
    This improved readability and made business rules easier to change.
    7. Added input validation
    Validation was added to prevent invalid data from entering the system.
    For Employee:
    • name must be a non-empty string
    • hourly_rate must be a non-negative number
    • hours_worked must be a non-negative number
    For PayrollSystem:
    • only Employee objects can be added
    This made the code safer and more robust.
    8. Added code comments
    Comments were added to explain:
    • validation logic
    • overtime calculation
    • class responsibilities
    • payroll processing flow

    • Solution

    
# Refactored OOP version with input validation and code comments

class Employee:
    STANDARD_HOURS = 40
    OVERTIME_MULTIPLIER = 1.5

    def __init__(self, name, hourly_rate, hours_worked):
        # Validate employee name
        if not isinstance(name, str) or not name.strip():
            raise ValueError("name must be a non-empty string")

        # Validate hourly rate
        if not isinstance(hourly_rate, (int, float)) or hourly_rate < 0:
            raise ValueError("hourly_rate must be a non-negative number")

        # Validate hours worked
        if not isinstance(hours_worked, (int, float)) or hours_worked < 0:
            raise ValueError("hours_worked must be a non-negative number")

        self.name = name
        self.hourly_rate = hourly_rate
        self.hours_worked = hours_worked

    def calculate_pay(self):
        # Apply overtime rule when standard hours are exceeded
        if self.hours_worked > self.STANDARD_HOURS:
            overtime_hours = self.hours_worked - self.STANDARD_HOURS
            regular_hours = self.STANDARD_HOURS
            return (
                regular_hours * self.hourly_rate
                + overtime_hours * self.hourly_rate * self.OVERTIME_MULTIPLIER
            )

        # Standard pay calculation
        return self.hourly_rate * self.hours_worked

    def print_payslip(self):
        # Display employee payroll details
        print(f"Employee: {self.name}")
        print(f"Hours Worked: {self.hours_worked}")
        print(f"Hourly Rate: {self.hourly_rate}")
        print(f"Gross Pay: {self.calculate_pay()}")


class PayrollSystem:
    def __init__(self):
        # Store all employees managed by the payroll system
        self.employees = []

    def add_employee(self, employee):
        # Ensure only Employee objects are added
        if not isinstance(employee, Employee):
            raise TypeError("employee must be an instance of Employee")

        self.employees.append(employee)

    def print_all_payslips(self):
        # Print payslips for every employee in the system
        for employee in self.employees:
            employee.print_payslip()


# Example usage
payroll = PayrollSystem()
payroll.add_employee(Employee("Alice", 25, 45))
payroll.add_employee(Employee("Bob", 30, 35))
payroll.print_all_payslips()
