Unit 2: SOLID Principles of Object-Oriented Design

Unit 2 is about the SOLID principles, a set of five design principles that are essential for writing clean, maintainable, and scalable object-oriented code. These principles help design software systems that are easier to understand, extend, and modify.

Objectives

  • Understand and explain  each of the five SOLID principles and their importance in object-oriented design
  • Apply the SOLID principles to refactor code, making it more maintainable and scalable
  • Recognise how SOLID principles are used in real-world systems to improve software design.>

    Refactored Code Steps (SOLID Principles)

    Single Responsibility Principle (SRP):

    Problem: Order does too much.
    Solution: Split into:
    Order: Manages items and calculates totals.

    PaymentProcessor: Handles payments.

    Improvement: Order no longer handles payments.

    Open/Closed Principle (OCP):

    Problem: Adding a new payment method requires modifying PaymentProcessor.
    Solution: Use abstraction (e.g., an interface or abstract class).

    Liskov Substitution Principle (LSP):

    Problem: Subclasses should replace their parent class without breaking functionality.
    Solution: Ensure all PaymentMethod subclasses implement pay() correctly.
    Improvement: CryptoPayment can replace PaymentMethod without issues.

    Interface Segregation Principle (ISP):

    Problem: If PaymentMethod had unrelated methods, subclasses would be forced to implement them.
    Solution: Keep interfaces small and focused.
    Improvement: Classes only implement what they need.

    Dependency Inversion Principle (DIP):

    Problem: High-level Order depends on low-level PaymentProcessor.
    Solution: Depend on abstractions (PaymentMethod).
    Improvement: Order depends on PaymentMethod (abstraction), not concrete classes.

    Solution

    
from abc import ABC, abstractmethod

class PaymentMethod(ABC):
    @abstractmethod
    def name(self):
        pass
class CreditCardPayment(PaymentMethod):
     def name (self) ->str:
        return "Credit card"
        
class PaypalPayment(PaymentMethod):
    def name (self) ->str:
        return "Paypal"

class CryptoPayment(PaymentMethod):
     def name (self) ->str:
        return "Crypto"
    
     def fun2(self):
        return "fun2"


class PaymentProcessor:   
    def __init__(self, method: PaymentMethod) -> None:
        self.__pay_method = method.name()

    def pay(self):
        print(f"Processing {self.__pay_method} payment...")

class Order:
    def __init__(self, selected_payment: PaymentProcessor) -> None:
        self.items = []
