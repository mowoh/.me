Unit 7: Secure Coding Practices in Object-Oriented Programming

Explore Common Vulnerabilities in OO Systems:

  • Covered key techniques for writing secure code, including input validation, proper authentication, and exception handling

    • Objectives

  • Understand and explain the importance of secure coding in software development
  • Identify common vulnerabilities in object-oriented systems, such as injection attacks and buffer overflows.
  • Apply secure coding techniques, including input validation, proper authentication, and exception handling.
  • Refactor a codebase to eliminate security vulnerabilities through a practical exercise

    • Task

    A legacy codebase has multiple security vulnerabilities, including SQL injection, improper error handling, and weak authentication 
    
# Vulnerable code
def login(username, password):
    query = f"SELECT * FROM users WHERE username='{username}' AND password='{password}'"
    result = execute_query(query)
    if result:
        return "Login successful"
    else:
        return "Invalid credentials"

    Solution Steps

  • Analyse the codebase for common vulnerabilities (e.g., SQL injection, improper error handling).
  • Implement input validation and sanitisation
  • Replace weak authentication mechanisms with secure ones
  • Improve exception handling to avoid exposing sensitive information
  • Encapsulate sensitive data and methods
  • Test the refactored codebase to ensure vulnerabilities are eliminated

    • Solution

    
# Secure code
import bcrypt

def login(username, password):
    query = "SELECT * FROM users WHERE username=?"
    user = execute_query(query, (username,))
    if user and bcrypt.checkpw(password.encode(), user['password'].encode()):
        return "Login successful"
    else:
        return "Invalid credentials"
