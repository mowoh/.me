Unit 3: Design Patterns I - Creational Patterns

Focus on Five Key Creational Design Patterns:
 Singleton: Ensures a class has only one instance and provides a global point of access to it.
Factory Method: Defines an interface for creating objects but let's subclasses alter the type of objects created.
Builder: Separates the construction of a complex object from its representation.
Prototype: Allows cloning of objects without making the code dependent on their classes.
Abstract Factory: Provides an interface for creating families of related or dependent objects.

Objectives

  • Understand and explain what design patterns are and why they are important in software development.
  • Identify and apply key creational design patterns, including Singleton, Factory Method, Builder, Prototype, and Abstract Factory.
  • Recognise how creational patterns are used in real-world software systems to improve flexibility and maintainability.
  • Implement the Factory Method pattern in a practical coding exercise

    • Task

    You are tasked with designing a system for a car manufacturing company that produces different types of cars (e.g., Sedan, SUV, Hatchback).
    Using the Factory Method Pattern, implement a solution that allows the company to create cars without specifying the exact class of the car in the main program.

    Solution Steps

    A Car interface or abstract class with a method drive(). Concrete car classes (Sedan, SUV, Hatchback) that implement the Car interface. A CarFactory abstract class with a factory method create_car(). Concrete factory classes (SedanFactory, SUVFactory, HatchbackFactory) that override the create_car() method. A demonstration of how the factory method pattern is used to create and drive different types of cars..

    Solution

    
from abc import ABC, abstractmethod

"""
Topic: Design(Creational) Pattern - Factory Methods
"""
class Car(ABC):
    @abstractmethod
    def drive(self) -> None:
        pass

class SUV(Car):
    def drive(self) -> None:
        print("Driving an SUV...") 

class Sedan (Car):
    def drive(self) -> None:
        print("Driving a Sedan...")

class Hatchback(Car):
    def drive(self) -> None:
        print("Driving a Hatchback...")

''' Car factory '''
class CarFactory(ABC):
    @abstractmethod
    def CreateCar(self)-> Car:
        pass
class SuvFactory(CarFactory):
    def CreateCar(self)-> Car:
        return SUV()
class SedanFactory(CarFactory):
    def CreateCar(self)-> Car:
        return Sedan()
class HatchbackFactory(CarFactory):
    def CreateCar(self)-> Car:
        return Hatchback()

'''
Client side:
'''
class Client:
    def __init__ (self) -> None:
        self.factories = [SuvFactory(),SedanFactory(),HatchbackFactory()]
    
    def go(self):
        for factory in self.factories:
            car = factory.CreateCar()
            car.drive()

if __name__ == "__main__":
    
    client = Client()
    client.go()
